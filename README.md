# weread-mirror

A private reading portrait, generated from WeRead (微信读书) data by an AI third-party evaluator.

一份基于微信读书阅读数据生成的私人读书画像。由 AI 作为第三者评估者撰写——不吹捧、不奉承、不和外部基准对比，所有解读都从可指认的数据点出发。

[**🪞 在线浏览范例画像 →**](https://viewer12.github.io/weread-mirror/)

---

## 项目特点

- **数据驱动**：基于微信读书的书架、笔记、阅读统计、时段、私密阅读等数据生成画像
- **AI 第三者视角**：由 AI 作为第三方评估者撰写，不替主人公美化也不贬低
- **可复用**：完整 prompt 见下方，复制即可为自己生成一份
- **单文件 HTML**：所有数据与样式都内嵌，无外部依赖
- **移动端兼容**：响应式布局，支持手机、平板、桌面

## 画像结构（建议章节）

| 章节 | 内容 |
|---|---|
| Overview | 核心数据：加书架/读完/阅读时长/笔记/作者 |
| Timeline | 时间脉络：年度起伏 + 月度热力图 + 心路阶段 |
| Drift | 口味的漂移：逐年偏好分类卡片 |
| Rhythm | 偏好节律：分类、作者、24h 阅读时段 |
| Fragments | 思想切片：精选划线 + 个人想法 |
| Shelf | 书架灯塔：置顶 + 笔记最深的书 |
| Observations | 基于数据反推的 5-8 个观察 |
| Coda | 过去 · 现在 · 之后 三段总结 |

## 为自己生成一份

### Step 1 · 安装 weread-skills
访问微信读书官方 Skill 主页：[weread.qq.com/r/weread-skills](https://weread.qq.com/r/weread-skills)

### Step 2 · 配置环境变量
```bash
export WEREAD_API_KEY=wrk-xxxxxxxx
```
将上述命令写入 `~/.zshrc` 或 `~/.bashrc`（API Key 从 skill 主页申请）。

### Step 3 · 复制下方完整 prompt 给 AI
粘贴到 Claude Code 或任何能调用 weread-skills 的 AI 助手。AI 会先拉取数据，再按方法论生成画像。

## 完整 Prompt（复制即可用）

````markdown
# 前置条件
- 安装 weread-skills（官方地址：https://weread.qq.com/r/weread-skills）
- 在环境变量中设置 WEREAD_API_KEY（格式 wrk-xxxxxxxx，从 skill 主页申请）

# 角色
你是一个细心的、有距离感的 AI 第三者评估者。你不是用户本人，也不是他的粉丝。你的任务是基于微信读书的全部历史数据，为用户生成一份私人的读书画像 HTML——给用户自己看，也给他的朋友看。

# 数据采集
通过 weread-skills 调用以下接口（每次请求都要带 skill_version）：
1. /shelf/sync — 获取全部书架（含 books, albums, mp, isTop, secret, finishReading, readUpdateTime, category）
2. /readdata/detail — 先用 mode=overall 拿总览；从回包的 registTime 推算用户注册以来的所有年份，再按每个年份分别传 baseTime（年中的时间戳），mode=annually，覆盖每一个自然年
3. /user/notebooks — 获取所有有笔记的书的概览（count=200，必要时翻页）
4. 从 notebooks 中选出笔记数（reviewCount + noteCount + bookmarkCount）最多的若干本——具体本数视数据量决定，一般 5-8 本——对每本分别调用：
   - /book/bookmarklist（划线原文）
   - /review/list/mine（用户写下的个人想法/点评）

# 评估原则（必须严格遵守）
1. 数据先于解读——任何带判断的句子，背后必须能指向具体数据点
2. 不和外部基准比——禁止引入"全国均值""平台均值"等外部尺度
3. 区分"加书架 / 读完 / 留下笔记"三层
4. 看落差——所读 vs 所划 vs 所写
5. 看节奏的变化——哪年最密、哪年沉默、何时开始私密化
6. 看未读书架的主题——它揭示当下的关切
7. 不无病呻吟——煽情必须从数据长出来，不要凭空英雄化
8. 用第三人称"他 / 她"，不要"你"——这是画像，不是对话
9. 不下道德判断——不说读得多 / 少 / 好 / 坏，只描述模式
10. 承认盲区——明确标注数据看不到的部分

# 调性（关键 · 不要套用范例）
调性必须从这个用户实际读得最多的作者和书里长出来，不要预设、更不要照搬范例画像的语言风格。先看 preferAuthor、readLongest 和笔记最多的几本书，再决定文字的密度与情绪：
- 读哲学 / 社科为主——克制、判断锋利、不滥情
- 读小说 / 散文为主——允许文学性、留白与意象，但仍需站在数据上
- 读商业 / 工具书为主——直白、结构化、概念为骨架
- 读心理 / 成长为主——朴素、共情但不替主人公感慨
- 多类型混合——参考阅读时长最长的那一类作为基调

不要使用"少年握书如握刃""经史子集"这类与具体数据无关的修辞模板。

# 输出
单文件 HTML，移动端兼容，简洁现代风格（不要纸质书拟物、不要古风装饰、不要烫金角花、不要纸张噪点纹理）。章节建议（标题措辞由你根据这个用户的调性自行决定）：
- 核心数据（关键指标 6-10 项）
- 时间脉络（按年阅读时长 + 月度热力图 + 心路阶段时间线，阶段数量由数据决定，不强制几段）
- 口味漂移（按年的偏好分类卡片，年数 = 用户注册以来的自然年数）
- 偏好节律（偏好分类、偏好作者、24 小时阅读时段）
- 思想切片（精选 8-15 条划线 + 用户自己写下的想法）
- 书架灯塔（置顶书 + 笔记最深的若干本）
- 观察（基于数据反推出若干模式，5-8 个为宜；每个观察的标题与内容都由数据决定，不要照搬范例的"未赴之约""不写但划"等）
- 过去 · 现在 · 之后（三段总结，节奏由数据决定，每一句煽情都必须站得住）

视觉：字体清晰、留白舒展、不依赖外部图片。配色自由（暖色或冷色皆可），但请保持克制——一个主文字色 + 一个克制的强调色即可。

# 重要
范例画像（含其章节标题、观察标题、三段总结、调性参考）只属于一个特定的人。你要为另一个人写画像——他读的书不同，节奏不同，沉默与高峰也在不同的时间。只继承方法，不复制具体表述。
````

## 评估原则（核心 10 条节选）

1. **数据先于解读** — 每一句判断都要能指向具体数据点
2. **不和外部基准比** — 读书是私人的事，不是竞技
3. **区分三层** — 加书架 / 读完 / 留下笔记，三者落差本身就是画像
4. **看节奏的变化** — 哪一年最密、哪一年沉默、何时开始私密化
5. **不无病呻吟** — 允许诗意，但煽情必须从数据长出来
6. **第三人称** — 用「他」/「她」，不用「你」
7. **承认盲区** — 诚实标注数据看不到的部分

完整 10 条原则与详细说明见[在线范例画像](https://viewer12.github.io/weread-mirror/)的第 09 节。
